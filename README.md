# Shigetomi v2 ― チェックイン管理システム

## 施設概要

**宿 重富** ― 鹿児島県姶良市脇元1979（重富駅 徒歩5分）

一軒家を改装した旅館。1日1組限定・無人運営。
QRコードによるセルフチェックイン → Slackで管理者に通知。

**予約経路**: [楽天トラベル](https://travel.rakuten.co.jp/HOTEL/197826/197826.html) / Booking.com / じゃらんnet

---

## プロジェクト概要

Googleフォームベースのチェックインシステム（v1）を、独自フォームに全面移行するプロジェクト。

**v1 の課題**:
- パスポート写真アップロードにGoogleログインが必須
- 多言語UIが不完全（エラーメッセージ等がGoogleデフォルト）
- 言語選択ページとフォームのデザインが不統一

## 設計方針

- **JSON駆動のフォーム設計** — 質問項目は設定ファイルで管理。v2.2でGUIビルダーに発展
- **非エンジニア運用可能** — 日常業務はSlack通知だけで完結
- **セキュリティ** — パスポート写真は Private Bucket + 署名付きURL（有効期限付き）
- **コスト** — すべて無料枠内で運用（写真はフロント側リサイズで容量対策）
